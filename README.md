# 插件抽屉（plugin-hub）

把 Hana 顶栏插件整理成「顶栏常驻 + 下拉临时启动」的插件。

## 一拖即装

1. 打开 Hana 的插件管理界面。
2. 把 `plugin-hub-v1.0.0.zip` 拖进去安装。
3. 如果 Hana 提示 full-access 权限，请允许。该权限用于自动备份并修补 Hana 桌面端 `app.asar`。
4. 安装后重启 Hana。
5. 顶栏出现「抽屉」后即可使用。

首次安装时，插件会自动完成：

- 安装抽屉页面。
- 把「抽屉」默认置顶，避免入口丢失。
- 自动备份 Hana 的 `app.asar`。
- 自动写入增强补丁，让原生下拉菜单点击插件时只临时打开，不自动置顶。
- 在抽屉页面提示是否需要重启。

> 增强补丁写入后需要重启 Hana 才会生效，这是 Electron renderer 已经加载到内存导致的，无法完全省略。

## 功能

- 「顶栏」：常用插件固定显示在 Hana 顶栏。
- 「下拉」：低频插件收进 Hana 原生下拉菜单。
- 点击抽屉里的插件名称：直接跳转到对应插件页面，不改变置顶状态。
- 点击原生下拉菜单里的插件：增强补丁生效后，只临时打开，不自动置顶。
- 抽屉自身启动时自动置顶。

## 卸载与恢复

插件卸载时会自动尝试恢复自己改过的地方：

- 如果检测到插件目录已经被删除，才判断为真正卸载。
- 如果只是关闭 Hana、重载插件或禁用插件，不会恢复补丁，避免补丁反复失效。
- 真正卸载时会：
  - 使用安装时生成的备份恢复 `app.asar`。
  - 清理 preferences 里 `plugin-hub` 自己的 `hiddenTabs/tabOrder` 痕迹。
  - 不恢复或改动用户对其他插件的顶栏/下拉设置。

安全限制：

- 只有当前 `app.asar` 里含有本插件 marker 时才恢复。
- 只有备份文件里含有原始片段时才恢复。
- 缺少备份或 Hana 版本不匹配时会跳过，不强行覆盖。

恢复后同样需要重启 Hana 才能让桌面 renderer 使用恢复后的代码。

## 包结构

Hana 插件安装器支持 zip 内只有一个顶层目录。本包结构为：

```txt
plugin-hub/
  manifest.json
  index.js
  README.md
  routes/
    hub.js
```

## 注意

- 插件需要 `full-access`，因为增强补丁需要读写 Hana 安装目录下的 `resources/app.asar`。
- Hana 更新后 `app.asar` 可能被覆盖，插件启动时会重新自检并按需补丁。
- 如果未来 Hana 原生修复了“下拉点击自动置顶”的问题，本插件找不到旧片段时会跳过补丁，抽屉管理功能仍可正常使用。
